# 一、个人原版理解（原样保留）

在一号文件.C用EXPORT_SYMBOL(函数1或者变量1)暴露出来，然后在二号文件.H要声明函数1或者变量1，然后二号文件.C引用二号文件.H，最后就可以使用其他KO模块定义的函数和变量。

# 二、原版理解 纠错+标准规范版本（精准修正）

**你的整体逻辑完全正确，仅一处不规范细节修正：**

❌ 错误表述：在二号文件.h 声明函数/变量

✅ 标准规范：**谁提供符号，谁的头文件声明**

1. **提供方模块A（A.c）**：定义全局函数/变量 + `EXPORT_SYMBOL()` 导出符号

2. **提供方头文件（A.h）**：对该函数/变量做外部声明（给其他模块调用）

3. **使用方模块B（B.c）**：直接 `#include "A.h"`，即可调用A模块导出的函数/变量

4. 加载规则：**先加载提供方A.ko，再加载使用方B.ko**，否则符号找不到、加载失败

# 三、符号导出配套查看命令（全套）

## 1. 查看未加载 .ko 的导出符号

编译完成后，校验当前模块是否成功导出函数/变量

```bash
nm -g xxx.ko          # 查看所有导出符号
nm -g xxx.ko | grep T # 查看导出函数
nm -g xxx.ko | grep D # 查看导出变量

```

## 2. 查看已加载模块的导出符号

```bash
modinfo xxx.ko | grep Exports   # 查看模块导出列表
grep 符号名 /proc/kallsyms      # 在内核全局符号表查找

```

## 3. 两种导出宏区别

- **EXPORT_SYMBOL()**：所有模块均可调用

- **EXPORT_SYMBOL_GPL()**：仅协议为GPL的模块可调用，非GPL调用会内核告警

# 四、内核模块标准依赖体系（depmod 正规流程）

前面的 lsmod 仅能查看依赖，**真正生成、构建、管理内核模块依赖的核心命令是 depmod**，也是企业、考试、驱动开发标准流程。

## 1. 先查看本机内核版本（必须操作）

所有内核模块必须匹配当前内核版本，依赖文件存放在对应内核版本目录下

```bash
uname -r   # 查看当前运行内核版本
```

模块系统目录固定路径：`/lib/modules/内核版本号/`

## 2. 为什么要复制模块到 /lib/modules/版本目录？

1. 自己编译的 .ko 只是本地文件，内核系统**默认不识别、不纳入依赖管理**

2. 只有复制到系统内核模块标准目录，**depmod 才能扫描到该模块**，生成依赖关系

3. 后续可以使用 **modprobe 自动解析依赖、自动加载**

## 3. 完整部署流程

```bash
# 1. 复制自定义ko到系统内核模块目录（必须）
sudo cp xxx.ko /lib/modules/$(uname -r)/

# 2. 更新、生成模块依赖关系（核心命令 depmod）
sudo depmod -a

```

# 五、depmod 命令 完整原理（重点背诵）

## 1. 作用

扫描 `/lib/modules/内核版本/` 下所有 .ko 模块，**自动分析模块之间的依赖关系**，生成依赖数据库文件。

## 2. 生成的核心依赖文件

路径：`/lib/modules/$(uname -r)/modules.dep`

该文件记录了：**哪个模块依赖哪个模块**

## 3. depmod -a 参数含义

-a = all，扫描所有模块，全局更新所有依赖，开发中一律使用 `depmod -a`

# 六、基于depmod依赖的 标准加载/卸载模块流程

有了 depmod 生成的依赖表后，不再需要手动顺序加载，系统自动识别依赖。

## 1. 模块加载（两种方式）

### 方式1：insmod（手动加载，不解析依赖）

需要自己控制加载顺序，依赖模块必须手动先加载，否则报错

```bash
insmod A.ko
insmod B.ko

```

### 方式2：modprobe（自动解析依赖，推荐）

依靠 depmod 生成的依赖表，**自动加载所有依赖的父模块**

```bash
modprobe xxx   # 无需写.ko，自动找模块、自动补全依赖

```

## 2. 模块卸载（两种方式）

### 方式1：rmmod（手动卸载，不处理依赖）

被依赖模块无法直接卸载，必须先卸子模块

```bash
rmmod xxx

```

### 方式2：modprobe -r（递归卸载，推荐）

依靠depmod依赖表，**自动卸载该模块以及所有依赖它的子模块**

```bash
modprobe -r xxx
```
