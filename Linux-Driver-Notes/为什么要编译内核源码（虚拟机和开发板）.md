我看教程里面他是创建了一个在linuxdriver文件夹下创建的kernel文件夹，然后把内核源码下载进来编译，然后在linuxdriver文件夹下面开始写字符驱动的。他的这个kernel文件夹里面放内核源码有什么用，我那时候是直接创了一个文件夹，然后开始写字符驱动了。他说是为了内核模块的需要内核提供的各种底层接口，和我这样有什么区别。

之，这个问题问到根上了。你之前写 `hello_drv.ko` 的时候，实际上也在用内核源码，只是你没意识到。

---

## 你之前的做法

你在虚拟机上写驱动，Makefile 里有一行：

```makefile
KDIR := /lib/modules/$(shell uname -r)/build
```

这个 `/lib/modules/$(shell uname -r)/build` 就是一个指向**内核源码**的符号链接。它指向你虚拟机里安装的 `linux-headers-6.17.0-23-generic` 这个包，这个包里包含了内核编译模块所需的头文件和Makefile。

**你之前不是没用到内核源码，是虚拟机帮你装好了，你直接用就行。**

---

## 教程里为什么要自己下载编译内核源码

因为你的鲁班猫开发板是ARM架构的，它的内核和你虚拟机的x86内核完全不同。你要在开发板上加载自己写的驱动模块，这个模块必须是**针对开发板的内核版本编译**的。

教程里他下载内核源码，然后编译，目的就是拿到编译驱动模块所需要的**内核头文件、Makefile、配置信息**，这些必须和开发板上跑的内核**完全一致**。

---

## 两种做法的区别

| | 你之前在虚拟机上 | 教程里在开发板上 |
|------|------|------|
| **内核源码从哪来** | 虚拟机自带的`linux-headers`包 | 野火提供的开发板对应的内核源码 |
| **为什么能编译驱动** | 头文件+Makefile都在`/lib/modules/`下 | 需要自己编译内核，生成这些文件 |
| **编译出的`.ko`能跑在哪** | 只能跑在虚拟机（x86）上 | 只能跑在开发板（ARM）上 |
| **本质** | 都是利用内核源码提供的接口编译驱动模块 |

---

## 你现在该怎么做

**最简单的方法**：直接在你的鲁班猫开发板上编译驱动。

开发板刷好系统后，它本身就是一台Linux电脑。你先给它连上网，然后在开发板上安装内核头文件：

```bash
sudo apt update
sudo apt install linux-headers-$(uname -r) build-essential
```

然后你就可以直接在开发板上，用你之前那个熟悉的Makefile，编译你的 `hello_drv.ko`，然后 `insmod` 加载。**这和你虚拟机的操作一模一样。**

教程里他下载内核源码编译，是更底层的做法，适用于开发板没有现成的`linux-headers`包、或者需要修改内核配置的场景。你现在初学驱动，用开发板自带的包就够了。等你需要深入修改内核或写复杂驱动时，再用他那种方法。
