# 从物理地址层到二级指针参数层（小白易懂版）

1. 快递：这个快递类型是modbus_t类型的内存区

2. 单号：物理地址 *ctx

3. 存放单号的纸条：ctx

4. 存放纸条的抽屉：&ctx

5. 能写抽屉地址的纸条：**pp参数

这样一来，我代码里的东西就都很清楚了：

首先我编写了一个函数，操作的对象是main函数里的ctx指针。

函数内部需要用到ctx，所以**ctx就相当于找了一张能写抽屉地址的纸条，然后把ctx（存放单号的纸条）所在的抽屉地址写上去，这也是之后*ctx = bconnect的大前提。

说白了就是看需求：

你是想取快递，还是确认存单号的纸条是否有效？

取快递的话，只要找到*ctx对应的物理地址就好了。

假如你有一个函数，能返回一个可能有效、也可能无效的单号（物理地址），你就用同样的类型去接收这个返回值就好（也就是*ctx）。之后判断*ctx == NULL，就能知道这个单号（地址）有没有效了。
