- 异常清理唯一入口：为 atexit 终极清理函数提供完整资源清单

**你的理解完全正确（核心通俗精髓）**：该结构体最核心的作用，就是**收拢、替代所有散乱的全局变量**。不再单独定义一堆零散全局变量，所有资源、业务变量全部收纳在manager结构体内部，使用时统一通过「结构体变量名\.成员名」调用。最大两大实用优势：第一，资源统一规整、集中管理，代码结构清晰，杜绝全局变量杂乱乱象；第二，可直接对整个manager结构体执行memset内存清零，一键完成所有业务资源、状态变量的初始化清零，无需逐个手动清零零散变量，极简且零遗漏。

## 2\. 线程传参阶段（核心改造：彻底干掉全局变量依赖）

### 大白话

以前：四个线程各自捡全局变量用，杂乱无章。

现在：主线程创建一个**全局唯一工具箱**，把**工具箱指针**传给每一个子线程。

所有子线程：**只访问传进来的工具箱，不再访问任何全局变量**。

### 专业原理

通过 `pthread_create` 第四个参数传递统一上下文指针，实现**无全局变量多线程共享资源**，符合模块化、高内聚、低耦合工业编程规范，彻底杜绝全局变量竞争、乱改问题。

**你的理解完全正确（核心通俗版）**：第四个参数传递结构体指针，本质就是**所有子线程共用同一个工具箱，统一拿取、使用内部资源**，除了方便集中管理，还有多个工业级核心作用，同时彻底解答指针强制转换、参数含义、线程传参的底层逻辑。

### 一、第四个参数传结构体指针：除了方便管理，额外核心工业优势

- **彻底消灭全局变量**：不再依赖全局变量共享数据、句柄、锁，代码模块化，任意拷贝线程函数、复用代码都不会冲突，适配工业项目迭代开发。

- **资源唯一性绝对保障**：所有线程操作的是**同一个结构体实例**，不存在多份副本数据错乱、多把锁重复初始化、多连接重复创建的问题，数据一致性百分百可控。

- **线程解耦、低耦合**：线程函数不再绑定固定全局变量，只依赖传入的上下文指针，后续新增线程、新增资源，只需在结构体扩展，不用改线程逻辑。

- **支持多实例拓展（工业可扩展性）**：后续可以创建多个manager结构体，实现多网关、多链路隔离运行，全局变量完全做不到这一点。

- **生命周期强绑定**：线程资源、业务数据、运行状态全部绑定同一个上下文，初始化、销毁、异常回收完全同步，不会出现资源和线程生命周期脱节。

### 二、逐字拆解：为什么线程第一行必须强制还原指针？

**1\. 系统底层限制（根本原因）**

Linux `pthread_create` 是系统通用接口，为了适配**所有类型的传参**（int、结构体、字符串、自定义变量），统一将传入的参数定义为 **void\* 万能空指针**。

**void\* 的致命特点**：只存内存地址，**没有任何数据类型信息**，程序不知道这块内存是int、数组还是你的gateway\_manager\_t结构体。

**2\. 强制转换代码逐行白话解析**

```Plain Text
gateway_manager_t *mgr = (gateway_manager_t *)arg;
```

- **arg**：pthread\_create第四个参数传过来的原始参数，类型是`void*`，仅保存工具箱的内存地址，无类型属性。

- **\(gateway\_manager\_t \*\)**：强制类型转换，告诉编译器：**这个void\*地址，本质就是一个gateway\_manager\_t结构体的地址**。

- **gateway\_manager\_t \*mgr**：定义一个和结构体匹配的专属指针，接收转换后的地址，让程序能够识别结构体内部的所有成员（线程、锁、句柄、数据）。

**3\. 不转换的后果（必报错）**

如果不做强制还原，编译器不认识void\*指向的结构体，无法使用 `arg->running`、`arg->tcp_ctx` 等成员，直接编译报错。

### 三、完整参数链路闭环（百分百吃透）

**完整链路补充（解答 g\_mgr 在哪里定义、是什么缩写）**

先说明：**g\_mgr 是全局结构体实例，全程手写定义，并非缩写**。

命名释义：

- **g** = global（全局的）

- **mgr** = manager（管理器的通用简写，行业标准缩写）

完整含义：**全局唯一的网关资源管理器实例**。

它的定义代码在整篇代码的结构体下方，是**整个程序唯一的工具箱实体**，位置明确：

```Plain Text

```

这行代码作用：**根据你定义的 gateway\_manager\_t 结构体模板，在全局内存中开辟一整块专属内存，创建出唯一的工具箱对象 g\_mgr**，所有资源全部存在这一块内存中。

**完整闭环参数链路（最终完整版）**：

1\. 主线程：通过代码定义全局唯一工具箱 **g\_mgr**（gateway\_manager\_t 类型实体，程序唯一资源容器）；

2\. 创建线程时：取 **\&g\_mgr**（全局工具箱的真实内存地址）传给 pthread\_create 第四个参数；

3\. 系统封装：Linux 线程接口统一将自定义结构体地址，强制转为通用万能指针 **void\* arg** 传入子线程；

4\. 子线程还原：通过强制类型转换 **\(gateway\_manager\_t \*\)arg**，把无类型的万能指针，还原成专属工具箱指针，解锁结构体所有成员，正常读写全部资源。

2\. 创建线程时：把 `&g_mgr`（结构体真实地址）传给pthread\_create第四个参数；

3\. 系统封装：参数被统一转为通用 `void* arg` 传入子线程；

4\. 子线程还原：强制转回 `gateway_manager_t *`，恢复结构体类型，正常读写所有资源。

### 四、一句话终极总结

传参就是：**主线程把唯一工具箱地址递给所有子线程**；强制转换就是：**子线程把系统屏蔽的万能指针，还原成自己能看懂的工具箱指针**，既能统一共用资源、方便管理，又实现了无全局变量、高内聚、可拓展的工业级代码架构。

# 三、终极清理函数 gateway\_cleanup 详解（工业级灵魂）

## 大白话解释

`atexit()` 就是系统的「临终管家」。

只要程序快要死了（正常退出、崩溃、段错误、Ctrl\+C），系统**一定会自动执行**这个清理函数。

清理函数只干一件事：**遍历工具箱，所有开过的连接全关掉、所有锁全销毁、所有资源全释放**。

## 专业原理

`atexit` 注册进程退出回调，覆盖**正常退出、异常退出、信号终止**绝大多数场景。统一释放文件句柄、同步原语、设备连接资源，实现**零资源泄漏**，保障7\*24h长期运行稳定性。

## 清理函数完整逻辑（逐行工业级安全判断）

```Plain Text
void gateway_cleanup(void)
{
    LOG_INFO("网关开始执行工业级资源清理...");
    gateway_manager_t *mgr = &g_mgr;

    // 1. 关闭Modbus TCP连接
    if (mgr->tcp_ctx != NULL) {
        modbus_close(mgr->tcp_ctx);
        modbus_free(mgr->tcp_ctx);
        mgr->tcp_ctx = NULL;
    }

    // 2. 关闭Modbus RTU串口连接
    if (mgr->rtu_ctx != NULL) {
        modbus_close(mgr->rtu_ctx);
        modbus_free(mgr->rtu_ctx);
        mgr->rtu_ctx = NULL;
    }

    // 3. 销毁互斥锁
    pthread_mutex_destroy(&mgr->data_mutex);
    pthread_mutex_destroy(&mgr->bus_mutex);

    LOG_INFO("网关资源清理完成，无资源泄漏！");
}

```

## 注册方式（main函数最开头）

```Plain Text
atexit(gateway_cleanup);
```

只要写这一句，**任何退出场景都会自动清理**。

**你的理解完全正确（最终核心精髓，100%到位）**：以前 Modbus 连接关闭、资源释放、内存 free 都是零散写在业务代码里，容易漏执行、异常退出完全不执行。现在统一把所有设备句柄关闭、内存释放、锁销毁的清理逻辑，全部收拢到唯一的 `gateway_cleanup()` 函数中，再通过 `atexit()` 全局注册。无论程序是正常退出、异常崩溃、Ctrl\+C 终止，系统都会自动兜底执行这一套完整的 free/close/destroy 逻辑，一次性跑完所有资源释放流程，**彻底杜绝人工遗漏、异常场景下的内存泄漏、句柄泄漏、锁资源残留**。

**工业级核心本质**：将分散的资源析构逻辑**集中归一化、自动化兜底**，告别业务代码手动释放的不确定性，实现资源释放100%可靠。

# 五、完整改造后、可直接编译运行的最终代码

特点：**彻底干掉所有业务全局变量、全线程传参、统一资源管理、自动异常清理、零泄漏**

```Plain Text
#include <pthread.h>
#include <stdio.h>
#include <unistd.h>
#include <string.h>
#include <stdlib.h>

#include"modbus_tcp.h"
#include"modbus_rtu.h"
#include"can.h"
#include"mqtt_huawei.h"
#include"log.h"
#include <linux/can.h>
#include <sys/socket.h>
#include"config.h"

// ====================== 工业级网关资源管理器 核心结构体 ======================
typedef struct {
    // 线程句柄
    pthread_t threads[4];

    // 通信句柄
    modbus_t *rtu_ctx;
    modbus_t *tcp_ctx;

    // 同步互斥锁
    pthread_mutex_t data_mutex;
    pthread_mutex_t bus_mutex;

    // 全局运行状态机
    bool running;

    // 业务数据缓存
    uint16_t regs[2];
    unsigned short rtu_data[64];
    float latest_temperature;
    float latest_humidity;

    // 重连计数
    int tcp_retry;
    int rtu_retry;
} gateway_manager_t;

// 全局唯一网关管理器实例（整个程序仅此一个）
static gateway_manager_t g_mgr;

// ====================== 资源初始化函数 ======================
static void gateway_manager_init(gateway_manager_t *mgr)
{
    memset(mgr, 0, sizeof(gateway_manager_t));

    // 初始化锁
    pthread_mutex_init(&mgr->data_mutex, NULL);
    pthread_mutex_init(&mgr->bus_mutex, NULL);

    // 开启运行开关
    mgr->running = true;

    // 初始化默认寄存器数据
    mgr->regs[0] = 1;
    mgr->regs[1] = 2;
}

// ====================== 工业级终极清理函数 ======================
void gateway_cleanup(void)
{
    LOG_INFO("【工业清理】执行网关全资源释放...");
    gateway_manager_t *mgr = &g_mgr;

    // 关闭TCP连接
    if (mgr->tcp_ctx) {
        modbus_close(mgr->tcp_ctx);
        modbus_free(mgr->tcp_ctx);
        mgr->tcp_ctx = NULL;
    }

    // 关闭RTU串口连接
    if (mgr->rtu_ctx) {
        modbus_close(mgr->rtu_ctx);
        modbus_free(mgr->rtu_ctx);
        mgr->rtu_ctx = NULL;
    }

    // 销毁锁资源
    pthread_mutex_destroy(&mgr->data_mutex);
    pthread_mutex_destroy(&mgr->bus_mutex);

    LOG_INFO("【工业清理】所有资源释放完成，零泄漏！");
}

// ====================== 所有线程函数（全部改用传参上下文） ======================
void *modbus_tcp_read(void *arg) {
    gateway_manager_t *mgr = (gateway_manager_t *)arg;
    while (mgr->running) {
        if (modbus_robust_read(&mgr->tcp_ctx, 0, 10, mgr->regs) == -1) {
            printf("TCP:重连失败，5秒后继续重试\n");
            sleep(5);
            continue;
        }

        for (int i = 0; i < 2; i++) {
            printf("reg[%d] = %d\n", i, mgr->regs[i]);
        }
        my_can_send(0x123, 2, mgr->regs);
        sleep(1);
    }
    return NULL;
}

void *modbus_rtu_read(void *arg) {
    gateway_manager_t *mgr = (gateway_manager_t *)arg;
    while (mgr->running) {
        if (modbus_rtu_robust_read(&mgr->rtu_ctx,0,2,mgr->rtu_data) == -1)
        {
            LOG_ERROR("RTU:重连失败,5秒后继续重试\n");
            sleep(5);
            continue;
        }
        for (int i = 0; i < 2; i++) {
            LOG_INFO("RTU_DATA[%d] = %d\n", i, mgr->rtu_data[i]);
        }
        pthread_mutex_lock(&mgr->bus_mutex);
        my_can_send(0x123, 2, mgr->rtu_data);
        pthread_mutex_unlock(&mgr->bus_mutex);
        sleep(1);
    }
    return NULL;
}

void *can_receive_pthread(void *arg) {
    gateway_manager_t *mgr = (gateway_manager_t *)arg;
    unsigned char data[8];
    int len;

    while (mgr->running) {
        if (can_receive(data, &len) == 0) {
            pthread_mutex_lock(&mgr->data_mutex);
            if (len >= 2) {
                mgr->latest_humidity = data[0];
                mgr->latest_temperature = data[1];
            }
            pthread_mutex_unlock(&mgr->data_mutex);
```
